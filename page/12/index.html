Hotel Eggs

Tags: eggs
Things
Tuesday, June 4th, 2013
Tweet

I don’t know why I was surprised to discover that the scrambled eggs featured in our hotel’s continental breakfast come from a carton of homogenized egg product. Or that they are prepared by pouring the egg product into a non-stick metal tray which is inserted into a table-top “oven”. The oven has two slots; one for eggs and one for warming up breakfast meats.

I didn’t think for a minute that there was a kitchen hidden away somewhere with a short order cook whipping up batches of eggs. I just didn’t expect to see eggs prepared in a broom closet. When the broom closet became the egg closet, the family of brooms and mops living there were probably displaced, scattered across the hotel, crying out for their loved ones.

Everything else at this continental breakfast is delivered. Why even make the eggs there? So they’re “fresh”? Those poor brooms.

Vhat do you vant to know?

I wonder when the first heat lamp was installed into the first hotel bathroom ceiling. Was it a sensation? Did they advertise on road-side billboards and on postcards in the lobby? Did people write home about it?

"Dear Edna, you're not going to believe this.â€¦"

I walk into the bathroom and turn on the heat lamp by mistake; I have accidentally interrupted an interrogation. Hopefully, not mine. A toilet is no place to be interrogated. Especially at this time of night. I’m pretty sure the Geneva Convention has some pretty clear language about that fact.

Sorry for the inconvenience

We’re enjoying this prolonged hotel experience because the city of Portland, a fire sprinkler maintenance company, and my upstairs neighbor conspires to destroy our bedroom and bathroom ceiling.

Remember kids, renter’s insurance is cheap.

The back half of our apartment was sealed off behind zippered plastic barriers while mighty fans, heaters, and HEPA filters attempted to extract moisture, lead paint particles, and mold. The bedroom and bathroom ceiling were forcibly removed and replaced. Hopefully we’ll be home again this week, though slightly disheveled and in need of a new mattress.

We will likely have to sweep the walls, vacuum the bathtub, and launder the floors. But I’ll be able to make eggs again.

Adding PHP include paths to a project managed by Composer

Tags: dev, php
Development
Monday, May 27th, 2013
Tweet

I was doing a little work on Archie’s Recipes; I found that I couldn’t use Composer to update my dependencies because I had manually hacked up the autoload.php file with a manual PHP include path configuration.

After a little digging I discovered that Composer supports an include path property for legacy projects (like Zend Framework 1.x and Smarty) which aren’t designed to use PHP autoloading. The property holds an array of include paths which get baked into the generated autoloader files. This way the include paths are available everywhere you include the autoloader. The feature is listed as deprecated, so be aware that you might not be able to use it forever.

Simply add the ‘include-path’ property at the top level of the JSON object:

{
    "require": {
        "silex/silex": "1.0.*@dev",
    },
    "autoload": {
        "psr-0": {
            "": "src/"
        }
    },
    "include-path": ["src/dumb-old-library/","src/other-dumb-library/"]
}

Simply create a JSON array of path names to whichever dumb libraries don’t support autoloading.

If you are adding this to an exiting project, be sure to run php composer.phar update to regenerate the autoloader files.

I’ve updated my article on adding Zend Search Lucene to a Silex project with this feature.
